# Start Here

## What Thirsty-101 is

Thirsty-101 is the entry point into my work and the thinking behind it.

It exists because the finished artifact is rarely enough to explain the system that produced it. Repositories, papers, standards, experiments, diagrams, and conversations are outputs. This repository documents the relationships among them.

## What it is not

This is not:

- a claim that every idea is complete;
- a replacement for the source repositories;
- a compressed marketing version of the work;
- a timeline rewritten to make development look cleaner than it was.

## How to read it

You can enter through whichever question brought you here:

- **What is he building?** Start with [Projects](PROJECTS.md).
- **How does it all connect?** Start with the [Mind Map](MIND-MAP.md).
- **Why was a particular direction chosen?** Start with [Decisions](DECISIONS.md).
- **What is still uncertain?** Start with [Open Questions](OPEN-QUESTIONS.md).
- **What does a term mean here?** Start with the [Glossary](GLOSSARY.md).

## Information states

Entries should identify which state their contents occupy:

- **Observation** — something directly encountered, measured, read, or recorded.
- **Interpretation** — what the observation may mean.
- **Hypothesis** — a proposition that has not yet been sufficiently tested.
- **Decision** — a chosen direction, including the reason and constraints.
- **Implementation** — something materially built or encoded.
- **Superseded** — retained for history but no longer authoritative.

## Source-of-truth rule

Thirsty-101 explains and connects the work. It does not silently replace it.

When an entry refers to a standard, implementation, paper, repository, experiment, or artifact, it should link to the authoritative source. If the source changes, the entry should either be updated or explicitly marked as historical.

## Intended result

A visitor should be able to move from a plain-language question to the relevant concepts, decisions, evidence, and implementations without needing private context first.
